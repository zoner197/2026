# Introduction
Scope:
- Year 11/12 student
- Help understand wellbeing patterns and cognitive load
Constraints:
- Apple watch
- Small screen
- Portable, wearable device
- Limited compute
Existing solutions:
## [Be Focused](https://www.google.com/search?q=Be+Focused&mstk=AUtExfBQJ_1puPu_EQHbgI1yK0pfhhiLF9SKh37b4bgIpUT1VUJ3KplW2ZAWu0haW6zX-DSJm4ZKLYigECweD0xofY8lV60nruR1OTzPOhjYfE_EzEig8aS8OKG_p1PLuuF_emm1ErC6L8BdgkUkYUxcwEmKfvSTnRn6JNmtDbguavBciMko-qgg4pPksq_FAUMz9ziE&csui=3&ved=2ahUKEwi-4OT6mOSSAxUqzjgGHedYNVQQgK4QegQIARAC)
Be Focused allows users to break their tasks up into discrete intervals with short breaks in-between, utilizing a pomodoro timer system.

### Pros
- Pomodoro timer
	- Helps focus
	- Helps manage time
	- Helps avoid burn out
		- Wellbeing
- Create and manage tasks
	- Add notes
	- Organize with tags and filters
	- Compatible with other apps
- Reports
	- Detailed report to check progress
- Sounds and notifications
	- Ticking sound
	- Alarm sounds
	- Interactive notifications
- Sync with all devices to avoid time waste
### Cons
- Complicated; user may waste time fiddling with app rather than actually working
- UI has too many sub-menus
- Notifications and sounds are too frequent; distracting
- Summaries are too hard to understand for the average person

Possible solutions:
Apple Watch app:
- Minimizes interaction
	- Avoid sub-menus
	- Avoid manual user-input
- Automatically estimate
	- Focus score
	- Stress index
- Provide summary
	- Visual
	- Short text
	- Easy to understand
- Recommends breaks based on physiological data
- Use haptic feedback instead of loud notifications on phone or alarm sounds
App responds to user physiology rather than force user to manage tasks and time manually

# The User Story
Required features:
- Monitor
	- Focus levels
	- Stress indicators
	- Breaks
	- Daily routines
- Return meaningful feedback that helps
	- Improve focus
	- Improve wellbeing
	- Manage time
- Works on wearable device (Apple Watch)
	- Perform calculations and comparisons on sensor data
	- Collect relevant data from user and device sensors
Required usability principles:
- Accessibility
	- High contrast UI
	- Large tap zones, due to small screen
	- Minimal text
	- Haptic feedback
- Effectiveness
	- Focus score in real time
	- Break suggestions from stress patterns
- Safety
	- Haptics, no loud alarm
	- Encourage user
	- Avoid stress
- Utility
	- Combine time tracking + physiological tracking
	- Useful during study sessions
- Learnability
	- Single main dashboard
	- Swipe-based, no sub menu
	- No complex UI
